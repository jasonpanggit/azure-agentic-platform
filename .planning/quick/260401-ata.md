# Quick Task: Chat Panel — Table Rendering & Resizable Width

## What

Two improvements to the chat drawer (`ChatDrawer.tsx`):

1. **Markdown table rendering already works** -- `ChatBubble.tsx` uses `react-markdown` + `remark-gfm`, and `globals.css` has full `.chat-prose table/th/td` styles. No code change needed for table formatting. Agents that return markdown tables (lines with `|`) will render as styled HTML tables automatically. **Verify this works end-to-end** and tweak styles if the tables overflow the narrow panel (add horizontal scroll).

2. **Resizable width** -- The drawer is hardcoded at `width: '420px'` in `ChatDrawer.tsx`. Add a left-edge drag handle so users can resize from 360px to 720px.

## Why

- Tables: Agent responses listing VMs/resources should be scannable. The rendering pipeline is already in place but may clip on narrow widths.
- Width: 420px is too narrow for tabular data and long agent responses. Users need to widen the panel when reading detailed output.

## Tasks

- [x] **1. Add table overflow handling** -- In `globals.css`, wrap `.chat-prose table` in `overflow-x: auto` container style (or add `overflow-x: auto; display: block;` to `.chat-prose table`). This ensures wide tables scroll horizontally within the bubble instead of breaking layout.

- [x] **2. Create `useResizable` hook** -- New file `services/web-ui/lib/use-resizable.ts`. A lightweight hook that returns `{ width, onMouseDown }`. Tracks mouse drag on X-axis, clamps between `MIN_WIDTH` (360) and `MAX_WIDTH` (720), persists last width to `localStorage` key `chat-drawer-width`.

- [x] **3. Wire resize handle into `ChatDrawer.tsx`**:
  - Import `useResizable` hook
  - Replace hardcoded `width: '420px'` with dynamic `width` from hook
  - Add a 4px-wide invisible drag handle `<div>` on the left edge of the drawer panel, positioned absolutely, with `cursor: col-resize`
  - On hover: show a subtle 2px blue line (`var(--accent-blue)`) for affordance

- [ ] **4. Smoke test** -- Open drawer, send "Show my virtual machines", confirm response renders as HTML table if agent returns markdown table. Drag left edge to resize. Refresh page, confirm width persists.

- **Low**: Resize handle z-index could conflict with backdrop overlay (z-40). Drag handle should be z-46 (above drawer's z-45).
- **Low**: `localStorage` access in SSR -- guard with `typeof window !== 'undefined'` check in hook initializer.
- **None**: Table rendering -- `react-markdown` + `remark-gfm` + existing `.chat-prose` CSS already handles GFM tables. Only adding overflow-x for narrow widths.
