# Quick Task: Deploy Images to Container Registry

**ID:** 260328-2ir
**Created:** 2026-03-28
**Status:** complete

---

## Problem

The platform has 11 Dockerfiles (7 agents + 4 services) but the CI/CD workflows for image deployment to ACR are incomplete:

1. **Missing build workflows for 2 services:** `teams-bot` and `web-ui` have individual build workflows (`web-ui-build.yml`) but `teams-bot` has no dedicated Docker push workflow (only test CI in `phase6-ci.yml`).
2. **No unified deploy workflow:** Individual per-service/per-agent workflows exist, but there's no single "deploy all images" workflow that can be triggered manually (e.g., for first-time ACR population or full redeployment). Currently, images only build when their specific path changes on push to `main`.
3. **`use_placeholder_image = true` default in Terraform:** The `agent-apps` module defaults to a placeholder image. After first image deployment, CI/CD should set this to `false` â€” but there's no workflow that bridges image push with Terraform apply.

## Scope

Create a unified "Deploy All Images" workflow and fill the missing teams-bot Docker push workflow. This gives operators a single button to populate ACR for initial deployment or full redeployment.

---

## Tasks

### Task 1: Create teams-bot Docker push workflow
**File:** `.github/workflows/teams-bot-build.yml`

- Follow exact pattern of `api-gateway-build.yml` and `web-ui-build.yml`
- Trigger on push to `main` when `services/teams-bot/**` changes
- Use reusable `docker-push.yml` with:
  - `image_name: services/teams-bot`
  - `dockerfile_path: services/teams-bot/Dockerfile`
  - `build_context: services/teams-bot/`
- Permissions: `id-token: write`, `contents: read`

### Task 2: Create unified "Deploy All Images" workflow
**File:** `.github/workflows/deploy-all-images.yml`

- Trigger: `workflow_dispatch` only (manual trigger for operators)
- Optional input: `image_tag` (default: `latest`, can override with git SHA)
- Build all 11 images in parallel using the reusable `docker-push.yml`:
  - **Agents (7):** base, orchestrator, compute, network, storage, security, sre, arc
  - **Services (4):** api-gateway, arc-mcp-server, teams-bot, web-ui
- Agent base image must build first; all other agent images depend on it
- Service images are independent and can build in parallel with agent images
- Include a final summary job that lists all built images and their tags

### Task 3: Verify all Dockerfiles build correctly
- Dry-run validate each Dockerfile exists and has correct paths
- Verify the `docker-push.yml` reusable workflow `build_context` paths align with each Dockerfile's expected context
- Confirm `agent-apps/main.tf` image naming convention (`agents/<name>` / `services/<name>`) matches workflow `image_name` inputs
- Cross-reference: Terraform expects `${acr_login_server}/agents/orchestrator:${image_tag}` â€” ensure workflow produces that exact tag
