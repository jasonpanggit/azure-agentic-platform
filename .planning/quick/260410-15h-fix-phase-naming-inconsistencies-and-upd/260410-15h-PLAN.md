# Quick Plan: Fix Phase Naming Inconsistencies and Update Memory

**Date:** 2026-04-10
**Mode:** quick
**Status:** ready

---

## Problem Summary

GSD tooling counts plans vs. summaries to determine phase completion status. Three categories of naming drift cause false "in-progress" reports for phases that are actually done:

| Category | Phases | Issue |
|---|---|---|
| Old PLAN-NN-NN prefix | 12, 13 | `PLAN-12-01.md` instead of `12-01-PLAN.md`; `PLAN-12-01-SUMMARY.md` instead of `12-01-SUMMARY.md` |
| Duplicate plans | 13 | Both `13-01-PLAN.md` AND `PLAN-13-01.md` exist; `SUMMARY.md` orphan |
| SUMMARY-NN-NN prefix (reversed) | 05 | Summaries named `SUMMARY-05-00.md` instead of `05-00-SUMMARY.md` |
| Abbreviated summary names | 21 | Summaries named `21-1-SUMMARY.md` but plans named `21-1-terraform-activation-PLAN.md` (no description in summary name) |
| Phase-level summaries only | 22–26 | One `NN-SUMMARY.md` covers all plans; no per-plan `NN-N-*-SUMMARY.md` stub files |

## Tasks

### Task 1 — Fix Phase 12: Rename PLAN-12-NN files to canonical pattern

**Files to create** (canonical aliases pointing at existing content):
- `.planning/phases/12-eol.../12-01-SUMMARY.md` → stub referencing `PLAN-12-01-SUMMARY.md`
- `.planning/phases/12-eol.../12-02-SUMMARY.md` → stub referencing `PLAN-12-02-SUMMARY.md`
- `.planning/phases/12-eol.../12-03-SUMMARY.md` → stub referencing `PLAN-12-03-SUMMARY.md`

**Approach:** The existing `PLAN-12-NN.md` and `PLAN-12-NN-SUMMARY.md` files contain the real content — leave them in place (other tools may reference them). Create three thin canonical stub files `12-01-SUMMARY.md`, `12-02-SUMMARY.md`, `12-03-SUMMARY.md` that note "see PLAN-12-NN-SUMMARY.md for full content". This makes plan count (3) = summary count (3) in the canonical pattern.

**Acceptance:** `ls .planning/phases/12-*/` shows `12-01-SUMMARY.md`, `12-02-SUMMARY.md`, `12-03-SUMMARY.md` present alongside existing files.
