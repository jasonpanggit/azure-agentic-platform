## Tasks

### Task 1 — VMSS list: enrich `health_state` from Resource Health API
**File:** `services/api-gateway/vmss_endpoints.py`

- Add import shim after existing imports:
  ```python
  try:
      from azure.mgmt.resourcehealth import ResourceHealthMgmtClient as _RHClient
  except ImportError:
      try:
          from azure.mgmt.resourcehealth import MicrosoftResourceHealth as _RHClient
      except ImportError:
          _RHClient = None  # type: ignore[assignment,misc]
  ```
- Add helper `_get_vmss_health_states(resource_ids, credential)` modelled on `vm_inventory.py:_get_health_states_sync()` (lines 203–237):
  - Group resource IDs by subscription (extract sub from resource ID)
  - Instantiate `_RHClient` per subscription, call `availability_statuses.get_by_resource(resource_uri=rid)` per resource
  - Return `dict[resource_id_lower, availability_state]` (e.g. `"available"`, `"unavailable"`, `"unknown"`)
  - Guard entire function: return `{}` if `_RHClient is None` or any exception
- After ARG list builds `vmss_list`, run `_get_vmss_health_states` in executor (same executor pattern used for metrics)
- Update each item: `health_state = health_map.get(item["id"].lower(), "unknown")`
- Derive `power_state`: `"Running"` if `health_state == "available"` else `health_state.capitalize()`

### Task 2 — VMSS metrics: replace stub with real Azure Monitor calls
**File:** `services/api-gateway/vmss_endpoints.py`

- Locate stub at lines 294–311 (`GET /api/v1/vmss/{id}/metrics`)
- Replace with real implementation copying the pattern from `vm_detail.py:_fetch_single_metric()` (lines 291–339) + concurrent gather (lines 386–403):
  - Extract `subscription_id` from `vmss_id` using `_extract_subscription_id()`
  - Define VMSS metric names:  
    `["Percentage CPU", "Network In Total", "Network Out Total", "Disk Read Bytes", "Disk Write Bytes"]`
  - For each metric name, run `_fetch_single_metric(credential, subscription_id, vmss_id, metric_name, timespan, interval)` concurrently via `asyncio.gather` + `loop.run_in_executor`
  - Return shape: `{"resource_id", "timespan", "interval", "metrics": [{"name", "unit", "timeseries": [{"timestamp", "average", "maximum", "minimum"}]}]}`
  - On error (SDK unavailable, 404, etc.) return `{"resource_id": vmss_id, "timespan": timespan, "interval": interval, "metrics": []}`

### Task 3 — AKS metrics: replace stub with real Azure Monitor calls

  ```python
  try:
      from azure.mgmt.containerservice import ContainerServiceClient as _AKSClient
  except ImportError:
      _AKSClient = None  # type: ignore[assignment,misc]
  ```

- In the list endpoint (lines 220–235), after fetching cluster objects, call `aks_client.agent_pools.list(resource_group, cluster_name)` per cluster

- Fall back to `cluster.agent_pool_profiles`-based count if `_AKSClient is None` or exception
