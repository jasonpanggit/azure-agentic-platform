## What Was Done

Four previously-stubbed data paths in the VMSS and AKS API endpoints were replaced with real Azure SDK calls.

### Task 1 — VMSS list `health_state` from Resource Health API
**File:** `services/api-gateway/vmss_endpoints.py`

- Added `_RHClient` import shim (tries `ResourceHealthMgmtClient`, falls back to `MicrosoftResourceHealth`, then `None`).
- Added `_get_vmss_health_states(resource_ids, credential)` — groups IDs by subscription, calls `availability_statuses.get_by_resource()` per VMSS, returns `dict[rid.lower() → state]`. Returns `{}` if SDK unavailable or on any error.
- After the ARG list query builds `vmss_list`, the helper runs in `asyncio.get_event_loop().run_in_executor(None, ...)`.
- Each item's `health_state` is updated from the map; `power_state` derived as `"Running"` when `available`, `state.capitalize()` otherwise.

### Task 2 — VMSS metrics: real Azure Monitor calls
**File:** `services/api-gateway/vmss_endpoints.py`

- Added `_VMSS_METRIC_NAMES = ["Percentage CPU", "Network In Total", "Network Out Total", "Disk Read Bytes", "Disk Write Bytes"]`.
- Added `_fetch_single_metric(client, resource_id, metric_name, timespan, interval)` — verbatim pattern from `vm_detail.py`; returns `{name, unit, timeseries}` or `None` on error/empty.
- `get_vmss_metrics` endpoint now creates a `MonitorManagementClient`, fans out all 5 metrics via `asyncio.gather` + `run_in_executor`, filters `None` results, returns full response shape.
- Falls back to `metrics: []` on any exception.

### Task 3 — AKS metrics: real Azure Monitor calls
**File:** `services/api-gateway/aks_endpoints.py`

- Added `_AKS_METRIC_NAMES = ["Percentage CPU", "Network In Total", "Network Out Total", "node_cpu_usage_percentage", "node_memory_rss_percentage"]`.
- Added identical `_fetch_single_metric` helper for AKS.
- `get_aks_metrics` endpoint replaces the stub with the same concurrent gather pattern.
- AKS-specific metrics (node_cpu_usage_percentage etc.) surface as empty timeseries if the cluster doesn't emit them — graceful degradation per plan.

### Task 4 — AKS `ready_node_count` via `agent_pools.list()`
**File:** `services/api-gateway/aks_endpoints.py`, `services/api-gateway/requirements.txt`

- Added `_AKSClient` import shim (`ContainerServiceClient`, or `None`).
- In `get_aks_detail`, replaced `cluster.agent_pool_profiles` loop with `aks_client.agent_pools.list(resource_group, cluster_name)`.
- `ready_node_count` per pool = `pool.count` only when `pool.provisioning_state == "Succeeded"` and `pool.power_state.code == "Running"`, else `0`.
- Full fallback to `agent_pool_profiles`-based count if `_AKSClient is None` or `agent_pools.list()` raises any exception.
- `azure-mgmt-containerservice>=21.0.0` added to `requirements.txt` (was previously only a transitive dep).
