# Quick Plan: Make PatchDetailPanel Resizable by Dragging Left Edge

**Task ID:** 260407-9ep
**Type:** quick (UI enhancement)
**Branch:** `quick/260407-9ep-resizable-patch-panel`

---

## Context

`InstalledPatchesPanel` is a fixed right-side drawer (`fixed inset-y-0 right-0, max-w-2xl`). It already has a drag-to-reposition feature via the header. We need to add a **resize handle on the left edge** so the user can drag to widen/narrow the panel.

### Existing Pattern

`lib/use-resizable.ts` already implements this exact pattern for the chat drawer:
- `mousedown/mousemove/mouseup` on `window`
- `useRef` for drag state (no re-renders during drag)
- `col-resize` cursor + `user-select: none` on `document.body` during drag
- `localStorage` persistence on `mouseup`
- Dragging left increases width (right-side drawer)

### Approach

**Generalize `use-resizable.ts`** to accept config (min, max, default, storageKey) and reuse it in `InstalledPatchesPanel`. This avoids code duplication and keeps the hook DRY.

---

## Tasks

### Task 1: Generalize `use-resizable.ts` to accept config

**File:** `services/web-ui/lib/use-resizable.ts`

- Add a `UseResizableOptions` interface: `{ minWidth, maxWidth, defaultWidth, storageKey }`
- Change `useResizable()` to `useResizable(options?: UseResizableOptions)`
- Default to current values (`MIN=360, MAX=800, DEFAULT=420, KEY='chat-drawer-width'`) so the existing chat drawer call site is unchanged (no breaking change)
- Return type stays `{ width, onMouseDown }`

**Acceptance:**
- Existing chat drawer usage (`useResizable()` with no args) works identically
- `useResizable({ minWidth: 480, maxWidth: 1200, defaultWidth: 672, storageKey: 'patch-panel-width' })` returns correct width and persists

### Task 2: Add resize handle to `InstalledPatchesPanel`

**File:** `services/web-ui/components/InstalledPatchesPanel.tsx`

Changes:
1. Import `useResizable` from `@/lib/use-resizable`
2. Call `useResizable({ minWidth: 480, maxWidth: 1200, defaultWidth: 672, storageKey: 'patch-panel-width' })`
3. Replace `w-full max-w-2xl` on the dialog div with `style={{ width: panelWidth }}`
4. Add a resize handle `<div>` as the **first child** inside the dialog (left edge):
   - `absolute left-0 top-0 bottom-0 w-1.5 cursor-col-resize z-10 hover:bg-primary/20 active:bg-primary/30 transition-colors`
   - `onMouseDown={resizeOnMouseDown}`
   - Must NOT trigger the header drag-to-reposition
5. Ensure the existing header drag-to-reposition still works (it uses a separate `handleDragMouseDown` on the header div)
6. Panel width persisted to `localStorage` under key `patch-panel-width`; survives close/reopen

**Acceptance:**
- Dragging left edge left widens the panel; dragging right narrows it
- Width constrained to 480px - 1200px
- Width survives panel close + reopen
- Header drag-to-reposition still works
- `npx tsc --noEmit` passes
- `npm run build` passes

### Task 3: Verify

- `npx tsc --noEmit` exits 0
- `npm run build` exits 0
- Manual visual check: resize handle visible on hover, drag works
