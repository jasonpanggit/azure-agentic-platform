# Plan: Show Connection Status Instead of Power State for Arc VMs
**ID:** 260409-00u  
**Mode:** quick  
**Status:** ready

## Problem

Arc VMs show "Unknown" in the Power State column because:
1. **Backend**: `_normalize_power_state()` doesn't recognize "Connected"/"Disconnected" — returns "unknown" for both
2. **Frontend**: `PowerStateBadge` has no config entries for "connected"/"disconnected" states — falls through to the generic `{ label: state, color: 'var(--text-muted)' }` fallback (raw capitalized string, grey)

## Tasks

### Task 1: Backend fix — `services/api-gateway/vm_inventory.py`

**Location:** Lines 424–428 (Step 4: Build response loop)

**Change:** Before calling `_normalize_power_state`, check `vm_type`. For Arc VMs, normalize the connection status directly instead.

```python
# Current (lines 427–428):
power_raw = row.get("powerState", "")
power_state = _normalize_power_state(power_raw)

# Replace with:
power_raw = row.get("powerState", "")
vm_type = row.get("vmType", "Azure VM")
if vm_type == "Arc VM":
    raw_lower = power_raw.lower()
    if raw_lower == "connected":
        power_state = "connected"
    elif raw_lower == "disconnected":
        power_state = "disconnected"
    else:
        power_state = "unknown"
else:
    power_state = _normalize_power_state(power_raw)
```

Note: `_normalize_power_state` is unchanged. The `vm_type` read here is only used locally — the row's `vmType` value is also still passed to the response dict at line 447 (`row.get("vmType", "Azure VM")`), so no duplication issue.

---

### Task 2: Frontend fix — `services/web-ui/components/VMTab.tsx`

**Location:** `PowerStateBadge` function, lines 30–49

**Change:** Add `connected` and `disconnected` entries to the config lookup object.

```typescript
// Current config object:
const config = {
  running: { label: 'Running', color: 'var(--accent-green)' },
  stopped: { label: 'Stopped', color: 'var(--accent-yellow)' },
  deallocated: { label: 'Deallocated', color: 'var(--text-muted)' },
}[state.toLowerCase()] ?? { label: state, color: 'var(--text-muted)' }

// Replace with:
const config = {
  running: { label: 'Running', color: 'var(--accent-green)' },
  stopped: { label: 'Stopped', color: 'var(--accent-yellow)' },
  deallocated: { label: 'Deallocated', color: 'var(--text-muted)' },
  connected: { label: 'Connected', color: 'var(--accent-green)' },
  disconnected: { label: 'Disconnected', color: 'var(--accent-red)' },
}[state.toLowerCase()] ?? { label: state, color: 'var(--text-muted)' }
```

No column header change needed — "Power State" is an acceptable shared label for both VM types.
