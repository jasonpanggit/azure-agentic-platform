# Quick Plan: Show Resource Name at Top of Overview Tab in All Detail Panels

**ID:** 260413-1z2
**Created:** 2026-04-13
**Type:** UI Enhancement
**Branch:** `quick/260413-1z2-overview-resource-name`

## Problem

The resource name is only visible in the panel header (which also contains close/refresh buttons and tabs). When users are looking at the Overview tab content, the resource name is not prominently displayed at the top of the content area. This makes it harder to quickly identify which resource is being inspected, especially when the header name truncates on narrow panels.

## Current State

| Panel | Header shows name? | Overview tab shows name? |
|-------|-------------------|------------------------|
| VMDetailPanel | Yes (truncated) | No — jumps straight to stat cards |
| VMSSDetailPanel | Yes (truncated) | No — jumps straight to summary cards |
| AKSDetailPanel | Yes (truncated) | No — jumps straight to summary cards |
| PatchDetailPanel | Yes (h2 + badges) | N/A — no overview tab (pending/installed tabs) |

## Tasks

### Task 1: Add resource name heading to Overview tab in VMDetailPanel, VMSSDetailPanel, and AKSDetailPanel

**Files:** `services/web-ui/components/VMDetailPanel.tsx`, `VMSSDetailPanel.tsx`, `AKSDetailPanel.tsx`

For each panel, add a resource name section at the very top of the Overview tab content (before the stat cards grid), containing:
- Resource name as a prominent heading (`text-base font-semibold`)
- Resource type label underneath (e.g., "Virtual Machine", "Scale Set", "AKS Cluster") in `text-[11px]` muted text
- For VMDetailPanel: also show the `vm_type` (Azure VM / Arc VM) as a small badge next to the type label

**Pattern (same across all 3 panels):**
```tsx
{/* Resource name heading */}
<div className="mb-1">
  <h2 className="text-base font-semibold" style={{ color: 'var(--text-primary)' }}>
    {resourceName}
  </h2>
  <p className="text-[11px]" style={{ color: 'var(--text-muted)' }}>
    {resourceTypeLabel}
  </p>
</div>
```

Insert location:
- **VMDetailPanel:** Inside the `activeTab === 'overview'` block, after the opening `<div className="p-4 space-y-4">`, before the stat cards grid (line ~809)
- **VMSSDetailPanel:** Inside the `activeTab === 'overview'` block, after loading check resolves, before the summary cards grid (line ~438)
- **AKSDetailPanel:** Inside the `activeTab === 'overview'` block, after loading check resolves, before the summary cards grid (line ~464)

**PatchDetailPanel excluded** — it already shows `machine.machineName` prominently as an `<h2>` in its header section, and has no overview tab. No change needed.
