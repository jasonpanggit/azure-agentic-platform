### Task 1 — G-01: Wire `AZURE_MCP_SERVER_URL` in Terraform for patch and EOL agents ✅
**Commits:** `c2f1736`

**Problem:** `agents/patch/agent.py` and `agents/eol/agent.py` both read `AZURE_MCP_SERVER_URL` to mount MCPTool / MCPStreamableHTTPTool at startup. The variable was never declared in Terraform and never injected as a container env var. Both agents degrade gracefully (MCP skipped if URL empty) but silently — all Azure MCP tools in their `ALLOWED_MCP_TOOLS` lists were unreachable at runtime.

**Changes:**
- `terraform/modules/agent-apps/variables.tf` — added `azure_mcp_server_url` variable (type=string, default="") following the exact same pattern as `arc_mcp_server_url`
- `terraform/modules/agent-apps/main.tf` — added `dynamic "env"` block with `contains(["patch", "eol"], each.key) && var.azure_mcp_server_url != ""` condition, matching the arc pattern
- `terraform/envs/prod/terraform.tfvars` — added commented placeholder with instructions for operator
- `terraform/envs/staging/terraform.tfvars` — same
- `terraform/envs/dev/terraform.tfvars` — same

**Verification:** `terraform fmt -check` passes on all 5 modified files; `grep -r AZURE_MCP_SERVER_URL terraform/` finds variable declaration and dynamic env block.
