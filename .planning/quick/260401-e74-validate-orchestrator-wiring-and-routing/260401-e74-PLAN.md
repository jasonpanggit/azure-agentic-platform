### Task 1: Fix G-01 — Wire `AZURE_MCP_SERVER_URL` in Terraform for patch and EOL agents

**Files to change:**
- `terraform/modules/agent-apps/variables.tf` — add `azure_mcp_server_url` variable
- `terraform/modules/agent-apps/main.tf` — add two `dynamic "env"` blocks (patch + eol containers)
- `terraform/envs/prod/terraform.tfvars` — add `azure_mcp_server_url` placeholder (operator sets value)
- `terraform/envs/staging/terraform.tfvars` — add `azure_mcp_server_url` placeholder
- `terraform/envs/dev/terraform.tfvars` — add `azure_mcp_server_url` placeholder (empty default OK)

**Pattern to follow:** Exact same approach as `arc_mcp_server_url` (variables.tf line 133–137, main.tf lines 210–217).

**Detailed steps:**

1. **`variables.tf`** — add after the `arc_mcp_server_url` variable (line 137):
   ```hcl
   variable "azure_mcp_server_url" {
     description = "Public or internal URL of the Azure MCP Server (npx @azure/mcp@latest) for patch and EOL agents (AZURE_MCP_SERVER_URL)"
     type        = string
     default     = ""
   }
   ```

2. **`main.tf`** — add two `dynamic "env"` blocks after the arc MCP block (after line 217), before the web-ui block:
   ```hcl
   # Inject Azure MCP Server URL into patch and EOL agents
   dynamic "env" {
     for_each = contains(["patch", "eol"], each.key) && var.azure_mcp_server_url != "" ? [1] : []
     content {
       name  = "AZURE_MCP_SERVER_URL"
       value = var.azure_mcp_server_url
     }
   }
   ```
   Note: One combined `dynamic` block with `contains(["patch", "eol"], each.key)` is cleaner than two separate blocks — mirrors the `orchestrator`/`api-gateway` pattern used for agent IDs (main.tf line 180).

3. **`terraform.tfvars` (prod/staging/dev)** — add comment + placeholder:
   ```
   # Azure MCP Server URL — set to the Container App / external URL where @azure/mcp runs
   # Example: https://ca-azure-mcp-server-prod.<env>.azurecontainerapps.io
   # azure_mcp_server_url = ""
   ```
   Leave commented out in all envs so the default="" in variables.tf takes effect (graceful degradation preserved — agents skip MCP mount if URL is empty). Operator uncomments and sets value when Azure MCP Server Container App URL is known.

**Acceptance criteria:**
- `terraform fmt -check` passes on all modified files
- `terraform validate` passes (run against dev env)
- `grep -r AZURE_MCP_SERVER_URL terraform/` finds the variable declaration and the dynamic env block

### Task 2: Fix G-03 — Write `agents/orchestrator/README.md` with full routing flow

**What to document (all based on current code — no speculation):**

#### Sections:

2. **Request Flow** — end-to-end narrative:
   ```
   Web UI / Teams
       ↓  HTTP POST /api/v1/chat  (SSE stream)
   API Gateway (ca-api-gateway-prod)
       ↓  Foundry Responses API  (thread_id, agent_id=ORCHESTRATOR_AGENT_ID)
   Orchestrator Agent (ca-orchestrator-prod)  ← Foundry Hosted Agent
       ↓  classify_incident_domain tool (Python, if domain unclear)
       ↓  connected-agent tool call (Foundry wiring)
   Domain Agent  (one of 8: compute / network / storage / security / arc / sre / patch / eol)
       ↓  Azure MCP tools (Azure MCP Server or Arc MCP Server)
       ↓  Python @ai_function tools (ARG, Log Analytics, Resource Health, etc.)
   Response streams back up the chain → SSE to browser
   ```
