# Quick Plan: Add Structured Logging to All Agents

**ID:** 260401-nk7
**Type:** quick
**Description:** Add structured logging to all 8 domain agents for Azure Container App log visibility
**Status:** COMPLETE

## Problem

The orchestrator is the ONLY agent with proper `logging.basicConfig()` + `logger.info()` calls. All 8 domain agents (compute, network, storage, security, arc, patch, eol, sre) have zero logging — their `__main__` blocks directly call `from_agent_framework(create_*()).run()` with no startup/exit messages, no log level configuration, and no structured format. This makes them invisible in Azure Container App log streaming and Log Analytics.

## Approach

1. Create a shared `setup_logging()` helper in `agents/shared/logging_config.py` to centralise `basicConfig` call with a consistent format, `LOG_LEVEL` env var support, and machine-readable output suitable for Azure Monitor.
2. Wire `setup_logging()` + `logger.info()` calls into all 8 domain agents' `__main__` blocks and factory functions, matching the orchestrator pattern.
3. Refactor the orchestrator's `__main__` to use the shared `setup_logging()` too (DRY).

## Reference Pattern (orchestrator/agent.py)

```python
import logging
logger = logging.getLogger(__name__)

# In factory:
logger.info("create_orchestrator: initialising Foundry client")

# In __main__:
if __name__ == "__main__":
    logging.basicConfig(
        level=logging.INFO,
        format="%(asctime)s %(levelname)s %(name)s %(message)s",
    )
    logger.info("orchestrator: starting up")
    from azure.ai.agentserver.agentframework import from_agent_framework
    logger.info("orchestrator: creating agent and binding to agentserver")
    from_agent_framework(create_orchestrator()).run()
    logger.info("orchestrator: agentserver exited")
```

## Tasks

### Task 1: Create `agents/shared/logging_config.py`

**Files:** `agents/shared/logging_config.py` (NEW)

Create a single shared utility:

```python
"""Structured logging setup for all AAP agent containers."""
from __future__ import annotations

import logging
import os


def setup_logging(agent_name: str) -> logging.Logger:
    """Configure structured logging for an agent container.

    Reads LOG_LEVEL from environment (default: INFO).
    Format: timestamp level logger_name message — machine-readable for Azure Monitor.

    Args:
        agent_name: Short agent name for the root logger context (e.g., "compute").

    Returns:
        A Logger instance for the calling module.
    """
    level = os.environ.get("LOG_LEVEL", "INFO").upper()
    logging.basicConfig(
        level=getattr(logging, level, logging.INFO),
        format="%(asctime)s %(levelname)s %(name)s %(message)s",
    )
    return logging.getLogger(f"aiops.{agent_name}")
```

Key design decisions:
- Returns a logger so callers don't need a separate `logging.getLogger()` call
- `LOG_LEVEL` env var for runtime control (default INFO) — operators can set to DEBUG on a per-container basis in Container Apps
- Logger name prefix `aiops.{agent_name}` for easy filtering in Log Analytics
- Same format as orchestrator's existing `basicConfig` for consistency
- No new dependencies — stdlib `logging` only
