# Quick Task: Move run-mock.sh to scripts/ and update paths

**ID:** 260328-1ca
**Created:** 2026-03-28
**Status:** complete

## Context

`run-mock.sh` lives at the repo root. All other operational scripts live under `scripts/`. Moving it there improves consistency. The script uses several relative paths that assume CWD is the repo root:

1. `pip install -r services/api-gateway/requirements.txt` (line 54)
2. `cat > _aap_bootstrap.py` writes to CWD (line 61)
3. Inside the heredoc: `"services/api-gateway/__init__.py"` and `"services/api-gateway"` (lines 73-74)
4. `--reload-dir services/api-gateway` (line 94)
5. `_aap_bootstrap:app` uvicorn module reference (line 90)

**Key finding:** No other files in the repo reference `run-mock.sh` (grep confirmed).

## Approach

The cleanest approach is to `cd` to the repo root at the top of the script so all existing relative paths remain valid. This avoids fragile `../` prefixes throughout and ensures the bootstrap shim is written to the repo root (where uvicorn expects it).
