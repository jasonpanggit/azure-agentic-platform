# Quick Task: Remove VM Filter from Cost Tab — Show All Advisor Cost Recommendations

**ID:** 260413-v9x
**Date:** 2026-04-13
**Scope:** 3 files (backend endpoint + frontend component + tests)

## Problem

The Cost tab currently filters Azure Advisor Cost recommendations to only show VMs (`"virtualmachines" in impacted_field.lower()`). This hides valuable cost recommendations for storage accounts, SQL databases, App Services, and other resource types. The user wants ALL Azure Advisor Cost recommendations displayed.

## Changes

### Task 1: Generalize `vm_cost.py` backend endpoint (backend)

**File:** `services/api-gateway/vm_cost.py`

1. Remove the `impacted_field` VM filter (line 84: `if not rec.impacted_field or "virtualmachines" not in rec.impacted_field.lower(): continue`)
2. Keep the `category == "Cost"` filter unchanged
3. Rename response fields from VM-specific to generic:
   - `"vms"` array key -> `"recommendations"` (keep `"vms"` as deprecated alias for backward compat in proxy route)
   - `"vm_name"` -> `"resource_name"` (use `rec.impacted_value`)
   - Add `"resource_type"` field (from `rec.impacted_field`, e.g. `"Microsoft.Compute/virtualMachines"`)
4. Update docstring and module docstring to reflect all resource types
5. Keep existing fields: `resource_group`, `resource_id`, `current_sku`, `target_sku`, `estimated_monthly_savings`, `annual_savings`, `savings_currency`, `impact`, `description`, `last_updated`
6. The endpoint path stays `/api/v1/vms/cost-summary` (renaming the URL is a breaking change for the proxy route; not worth it for this quick task)

### Task 2: Update `CostTab.tsx` frontend component (frontend)

**File:** `services/web-ui/components/CostTab.tsx`

1. Update `CostVM` interface:
   - Rename to `CostRecommendation`
   - `vm_name` -> `resource_name`
   - Add `resource_type: string`
2. Update `CostSummaryResponse`:
   - `vms` -> `recommendations` (read from `data.recommendations ?? data.vms ?? []` for backward compat during rollout)
3. Update table columns:
   - "VM Name" -> "Resource"
   - Add "Resource Type" column (formatted nicely, e.g. strip `Microsoft.` prefix)
   - "Current SKU" / "Recommended SKU" stay (may be empty for non-VM recs)
4. Update header badge: `{count} VMs` -> `{count} resources`
5. Update empty state text: "No rightsizing recommendations" -> "No cost recommendations found"
6. Variable renames: `vms` state -> `recommendations`, `setVMs` -> `setRecommendations`

### Task 3: Update tests (tests)

**File:** `services/api-gateway/tests/test_vm_cost.py`

   - Still exclude the Performance rec
2. Update response field assertions: `data["vms"]` -> `data["recommendations"]`
3. Update field name assertions: `vm_name` -> `resource_name`, add `resource_type` check
4. Keep route collision regression test unchanged (path hasn't changed)
5. Keep SDK-missing and subscription_id-required tests (update field names only)
