# Fix Storage Agent Stub Tools — SUMMARY

**Date:** 2026-04-19
**Commit:** `fix(storage-agent): implement real Azure SDK calls replacing empty stub responses`

## What Changed

`agents/storage/tools.py` — all 3 stub tools replaced with real Azure SDK implementations.

### Additions

- **Imports:** `time`, `logging`, `os`, `datetime`, lazy imports for `MonitorManagementClient`, `LogsQueryClient`/`LogsQueryStatus`, `StorageSyncManagementClient`
- **Helper:** `_parse_timespan_hours()` — converts ISO 8601 duration strings (PT2H, P1D, etc.) to integer hours
- **`get_credential`** added to `shared.auth` import (was missing from original stub)
- **`_extract_subscription_id`** imported from `agents.shared.subscription_utils`

### Tool implementations

| Tool | SDK Used | Key Behavior |
|---|---|---|
| `query_storage_metrics` | `azure-mgmt-monitor` `MetricsOperations.list()` | Iterates `metric_names`, queries each with 5-min granularity, returns timeseries data |
| `query_blob_diagnostics` | `azure-monitor-query` `LogsQueryClient` | Queries `StorageBlobLogs` for HTTP 4xx errors + recent ops; uses `LOG_ANALYTICS_WORKSPACE_ID` env var; returns advisory note + KQL if workspace not configured |
| `query_file_sync_health` | `azure-mgmt-storagesync` `StorageSyncManagementClient` | Discovers resource group via subscription listing, queries server endpoints for health/last-sync; uses `AZURE_SUBSCRIPTION_ID` env var |

### Pattern followed

Matches `agents/compute/tools.py` exactly:
- `start_time = time.monotonic()` at function entry
- `duration_ms` recorded in both `try` and `except` blocks
- Structured error dicts returned on failure — tools never raise
- Lazy import guards (`if Client is None: raise ImportError(...)`)
