# Quick Task: 260416-bf5 — Generate VAPID Keys and Set as Env Vars

**Mode:** quick  
**Date:** 2026-04-16  
**Status:** PLANNED

## Context

`services/api-gateway/push_notifications.py` already implements a full VAPID-backed Web Push
notification service. It reads three env vars at runtime:

| Var | Usage |
|-----|-------|
| `VAPID_PUBLIC_KEY` | Returned by `GET /api/v1/notifications/vapid-public-key`; used by browser to subscribe |
| `VAPID_PRIVATE_KEY` | Signs outbound push payloads via `pywebpush` |
| `VAPID_EMAIL` | `sub` claim in VAPID JWT (`mailto:…`) |

None of these are set on `ca-api-gateway-prod` yet, so push dispatch is silently skipped
(`send_push_to_all: VAPID_PRIVATE_KEY not set`) and the public-key endpoint returns 503.

`.gitignore` already contains `.env.*` (line 20), so `.env.vapid` is automatically gitignored — no changes needed there.

---

## Tasks

### Task 1 — Generate VAPID key pair

```bash
npx web-push generate-vapid-keys --json
```

Capture the JSON output (`publicKey`, `privateKey`).  
Write the three values to `.env.vapid` at repo root:

```
VAPID_PUBLIC_KEY=<publicKey>
VAPID_PRIVATE_KEY=<privateKey>
VAPID_EMAIL=mailto:ops@aap.local
```

**Acceptance:** `.env.vapid` exists, contains all three vars, is NOT tracked by git
(`git status` shows it absent or in ignored list).

---

### Task 2 — Set secrets and env vars on `ca-api-gateway-prod`

Store the private key as a Container App secret (secrets are encrypted at rest), then
reference it as an env var. Public key and email are non-sensitive — set directly as
plain env vars.

```bash
# 1. Add VAPID_PRIVATE_KEY as a secret
az containerapp secret set \
  --name ca-api-gateway-prod \
  --resource-group rg-aap-prod \
  --secrets "vapid-private-key=<VAPID_PRIVATE_KEY>"

# 2. Set env vars — private key via secret ref, others plain
az containerapp update \
  --name ca-api-gateway-prod \
  --resource-group rg-aap-prod \
  --set-env-vars \
    "VAPID_PUBLIC_KEY=<VAPID_PUBLIC_KEY>" \
    "VAPID_PRIVATE_KEY=secretref:vapid-private-key" \
    "VAPID_EMAIL=mailto:ops@aap.local"
```

**Acceptance:** `az containerapp show` confirms all three env vars present; a new
revision is active (revision suffix increments); `/api/v1/notifications/vapid-public-key`
returns `{"vapidPublicKey": "..."}` with HTTP 200.

## Notes

- `.gitignore` already covers `.env.*` — no change needed.
- Private key stored as a Container App **secret** (encrypted) and referenced via
  `secretref:` — never exposed in plain-text env var list.
- VAPID keys are single-use per domain; do not regenerate unless rotating (rotation
  breaks all existing browser subscriptions).
- Gateway FQDN can be retrieved with:
  `az containerapp show -n ca-api-gateway-prod -g rg-aap-prod --query properties.configuration.ingress.fqdn -o tsv`
