## Summary

Add an EOL (end-of-life) date column to the VM tab by creating a new batch endpoint in the api-gateway that queries the existing `eol_cache` PostgreSQL table (with endoflife.date API fallback), a Next.js proxy route, and a new column in `VMTab.tsx`.

---

## Tasks

### Task 1: Backend — `POST /api/v1/vms/eol` endpoint + router registration

**Files to create/modify:**
- **CREATE** `services/api-gateway/eol_endpoints.py`
- **MODIFY** `services/api-gateway/main.py` (import + `app.include_router`)

**Implementation details:**

1. Create `eol_endpoints.py` with `APIRouter(prefix="/api/v1/vms", tags=["vms"])`:
   - `POST /eol` — accepts `{ "os_names": ["Windows Server 2025 Standard", ...] }`
   - Request model: `EolBatchRequest(os_names: list[str])`
   - Response model: `EolBatchResponse(results: list[EolResult])` where `EolResult(os_name: str, eol_date: Optional[str], is_eol: Optional[bool], source: Optional[str])`

2. `_parse_os_for_eol(os_name: str) -> tuple[str, str] | None`:
   - Strip "Standard"/"Datacenter"/"Essentials" suffixes
   - Match Windows Server years: 2025, 2022, 2019, 2016, 2012-r2, 2012, 2008-r2, 2008
   - Return `(product_slug, cycle)` e.g. `("windows-server-2025", "2025")` or `None`
   - Add Ubuntu pattern: `("ubuntu", "22.04")` from "Ubuntu 22.04 LTS"

3. For each OS name in the batch:
   - Parse with `_parse_os_for_eol()` → skip if `None` (return `eol_date=None, is_eol=None`)
   - Query `eol_cache` table via `asyncpg.connect(resolve_postgres_dsn())`:
     ```sql
     SELECT eol_date, is_eol, source FROM eol_cache
     WHERE product = $1 AND version = $2 AND expires_at > now()
     LIMIT 1
     ```
   - If cache miss → HTTP GET `https://endoflife.date/api/{product_slug}/{cycle}.json`
   - Parse response: `eol` field (date string or `true`/`false`), compute `is_eol` from date comparison
   - Insert cache row with 24h `expires_at`
   - Return result

4. DB access: use `resolve_postgres_dsn()` from `services.api_gateway.runbook_rag` (reuse existing pattern — per-request `asyncpg.connect()`, not a persistent pool)

5. Register in `main.py`:
   ```python
   from services.api_gateway.eol_endpoints import router as eol_router
   # ... at include_router block:
   app.include_router(eol_router)
   ```

**Acceptance criteria:**
- [ ] `POST /api/v1/vms/eol` returns correct EOL data for Windows Server OS names
- [ ] Cache miss falls back to endoflife.date API
- [ ] Unrecognized OS names return `eol_date: null, is_eol: null`
- [ ] No imports from `agents/eol/tools.py`
- [ ] Tool functions never raise — return structured error results

### Task 2: Frontend — proxy route + EOL Date column in VMTab

**Files to create/modify:**
- **CREATE** `services/web-ui/app/api/proxy/vms/eol/route.ts`
- **MODIFY** `services/web-ui/components/VMTab.tsx`

**Implementation details:**

1. **Proxy route** (`app/api/proxy/vms/eol/route.ts`):
   - Copy pattern from `app/api/proxy/vms/route.ts`
   - `POST` handler, forward body to `${getApiGatewayUrl()}/api/v1/vms/eol`
   - Use `buildUpstreamHeaders(request)` + `AbortSignal.timeout(15000)`
   - On failure: return `{ results: [] }` gracefully

2. **VMTab.tsx** changes:
   - Add `eolMap` state: `Record<string, { eol_date: string | null; is_eol: boolean | null }>`
   - After `setVMs(data.vms)` in `fetchVMs()`, extract unique `os_name` values, fire POST to `/api/proxy/vms/eol` with `{ os_names: [...] }`
   - Build lookup map keyed by `os_name`
   - Add "EOL Date" column header (after "OS" column)
   - Column cell rendering:
     - If `is_eol === true` → red "EOL" badge using `color-mix(in srgb, var(--accent-red) 15%, transparent)` + formatted date
     - If `eol_date` exists and `is_eol === false` → formatted date (e.g. "Oct 2026") in `var(--text-secondary)`
     - If unknown (`null`) → `"—"` in `var(--text-muted)`
   - Date formatting: `new Date(eol_date).toLocaleDateString('en-US', { month: 'short', year: 'numeric' })`

**Acceptance criteria:**
- [ ] Proxy route forwards POST to api-gateway and returns results
- [ ] EOL Date column appears in VM table between OS and Type columns
- [ ] Red "EOL" badge for past-EOL VMs using CSS semantic tokens (not hardcoded Tailwind colors)
- [ ] "—" shown for unrecognized OS names
- [ ] No console.log in production code
