## Context

Recent fixes on main need to be deployed to production:
- **Web UI**: VMDetailPanel improvements
- **API Gateway**: vm_inventory.py OS normalization fixes

Both services have GitHub Actions workflows that build via ACR private agent pool (`aap-builder-prod`) and deploy to Azure Container Apps.

## Pipeline Architecture

```
workflow_dispatch -> docker-push.yml (az acr build --agent-pool aap-builder-prod)
                  -> container-app-deploy.yml (az containerapp update + health check)
```

- **web-ui-build.yml** -> builds `services/web-ui` image -> deploys to `ca-web-ui-prod` (healthcheck: `/`)
- **api-gateway-build.yml** -> builds `services/api-gateway` image -> deploys to `ca-api-gateway-prod` (healthcheck: `/health`)
- Both use self-hosted runner (`[self-hosted, azure, linux]`)
- Both push to ACR with `github.sha` tag + `latest` tag
- Deploy job runs only on `main` branch (not PRs)

## Commands

```bash
# Trigger both workflows (can run in parallel)
gh workflow run web-ui-build.yml --ref main
gh workflow run api-gateway-build.yml --ref main

# Monitor runs
gh run list --workflow=web-ui-build.yml --limit=1
gh run list --workflow=api-gateway-build.yml --limit=1

# Watch specific runs (get run IDs from list above)
gh run watch <run-id>

# Post-deploy verification
curl -s https://<ca-api-gateway-prod-fqdn>/health
curl -s -o /dev/null -w '%{http_code}' https://<ca-web-ui-prod-fqdn>/
```

## Prerequisites

- ACR agent pool `aap-builder-prod` must be in `Succeeded` state
- GitHub secrets (AZURE_CLIENT_ID, AZURE_CLIENT_SECRET, AZURE_TENANT_ID, AZURE_SUBSCRIPTION_ID) configured
- GitHub vars (ACR_LOGIN_SERVER, AZURE_RESOURCE_GROUP, NEXT_PUBLIC_* for web-ui) configured
- Self-hosted runner available

## Risks

- **ACR agent pool not ready**: If `aap-builder-prod` is still provisioning, builds will fail. Check with `az acr agentpool show`.
- **Self-hosted runner offline**: Both workflows require `[self-hosted, azure, linux]` runner. If offline, jobs will queue indefinitely.
