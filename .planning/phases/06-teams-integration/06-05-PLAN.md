# Plan 06-05: Teams App Manifest + Integration Tests + Deployment Config

## Goal

Complete Phase 6 with the Teams app manifest, deployment configuration (env vars, Container App specs), integration test stubs for all 6 success criteria, a final CI verification pass, and a UI-SPEC correction to align card schemas with the Action.Execute migration. This plan closes the loop on all TEAMS-* requirements.

## Context

Plans 06-01 through 06-04 delivered the full Teams bot implementation:
- 06-01: Project scaffold, 4 card builders, notify endpoint, CI workflow
- 06-02: Bot Framework integration, conversational flow, proactive messaging
- 06-03: Api-gateway changes (thread continuation, pending approvals, notifier refactor)
- 06-04: Escalation scheduler, proactive card posting wiring

This plan adds:
1. Teams app manifest (`manifest.json` + icons) for org deployment
2. Environment variable documentation (`.env.example`)
3. Integration test stubs for all 6 Phase 6 success criteria
4. CI workflow update to include integration test markers
5. UI-SPEC update to replace residual `Action.Http` schemas with `Action.Execute`

```bash
# API Gateway Connection

# Teams Channel
TEAMS_CHANNEL_ID=                    # Default Teams channel ID for proactive card posting

# Escalation
ESCALATION_INTERVAL_MINUTES=15       # Minutes before unacted approval triggers reminder (default: 15)

# Server
PORT=3978                            # Express server port (Teams convention: 3978)

# Azure Identity (auto-resolved on Container Apps)
# AZURE_CLIENT_ID=                   # Set automatically by Container Apps managed identity
```
