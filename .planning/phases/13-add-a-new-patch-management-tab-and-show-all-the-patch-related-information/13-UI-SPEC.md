## Data Flow

### Fetch Pattern

```
1. PatchTab mounts or subscriptions change (via useEffect on subscriptions dep)
2. If subscriptions.length === 0 → show empty state (no fetch)
3. Fetch both endpoints in parallel:
   - GET /api/proxy/patch/assessment?subscriptions=sub1,sub2
   - GET /api/proxy/patch/installations?subscriptions=sub1,sub2
4. Both use Promise.all() for concurrent fetch
5. Each uses AbortSignal.timeout(15000) (matching existing proxy pattern)
6. On success: setAssessmentData / setInstallationsData, setError(null)
7. On error: setError(message), data remains stale (or null on first load)
```

### State Variables

```typescript
const [assessmentData, setAssessmentData] = useState<AssessmentMachine[] | null>(null)
const [installationsData, setInstallationsData] = useState<Installation[] | null>(null)
const [loading, setLoading] = useState(true)
const [error, setError] = useState<string | null>(null)
const [complianceFilter, setComplianceFilter] = useState('all')
const [machineSearch, setMachineSearch] = useState('')
```

## Interaction Contracts

### Tab Activation

- Data fetches ONLY when `activeTab === 'patch'` (lazy load, per D-11)
- First activation triggers initial fetch
- Subsequent activations show cached data (no re-fetch unless Refresh clicked or subscriptions change)

### Refresh Button

- Label: "Refresh" (text) with `RefreshCw` icon (h-4 w-4)
- Variant: `outline`, size: `sm`
- On click: calls `fetchData()` manually
- Disabled during loading (`disabled={loading}`)
- Icon rotates during loading: add `animate-spin` class to `RefreshCw` when `loading === true`

### Subscription Scope

- Reads `selectedSubscriptions` from `useAppState()` (passed as `subscriptions` prop)
- When subscriptions change: clear data, set loading, re-fetch both endpoints
- Pattern: `useCallback(fetchData, [subscriptions])` + `useEffect(() => { fetchData() }, [fetchData])`

### Compliance Filter (Assessment Table)

- Client-side filter on loaded `assessmentData`
- Select options: `All` (value: `'all'`), `Compliant`, `NonCompliant`, `Unknown`
- When changed: filtered view updates immediately (no fetch)

### Machine Search (Assessment Table)

- Client-side filter on `name` field (case-insensitive includes match)
- Debounce: none needed (client-side, instant filter on small datasets)
- Combined with compliance filter (both must match)

### Table Row Hover

- `hover:bg-muted/30 transition-colors` (matching `ResourcesTab.tsx` and `AuditLogViewer.tsx` pattern)
- No row click action in this phase (KB drill-down deferred)

### Horizontal Scroll on Wide Tables

- Assessment table has 13 columns — will overflow on narrow viewports
- Wrapper: `overflow-x-auto` on the `rounded-md border` div containing the Table
- Table: `w-full min-w-[1000px]` (forces horizontal scroll below 1000px viewport width)

## Loading & Error States

### Loading State (Initial)

```
Section 1: Header with "Patch Management" + disabled Refresh button (with spinning icon)
Section 2: 5 skeleton cards in grid (each: h-[88px] rounded-lg matching MetricCard dimensions)
Section 3: "Patch Assessment" heading + 6 Skeleton rows (h-10 w-full, gap-1)
Section 4: "Installation History" heading + 6 Skeleton rows (h-10 w-full, gap-1)
```

### Error State

```
Section 1: Header with "Patch Management" + enabled Refresh button
Section 2: Alert variant="destructive" with error message
Sections 3-4: Not rendered when error is present and data is null
```

### Partial Data (one endpoint fails)

```
- If assessment fails but installations succeed: show Alert for assessment error, show installations table
- If installations fail but assessment succeeds: show assessment table + summary cards, show Alert for installations error
- Summary cards derive from whichever data is available (Failed Installs card shows "—" if installations data is null)
```

### Empty State (no subscriptions selected)

```
Full-width centered empty state:
  ShieldCheck icon (h-8 w-8, text-muted-foreground)
  "No patch data available" (font-semibold text-base text-foreground)
  "Patch assessment data will appear here..." (text-sm text-muted-foreground text-center max-w-[360px])
  py-16 vertical padding (matching ResourcesTab and ObservabilityTab empty states)
```

## API Contract (Proxy Routes)

### `GET /api/proxy/patch/assessment`

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `subscriptions` | string (comma-separated) | yes | Subscription IDs to query |

**Response:** `{ machines: AssessmentMachine[], total_count: number, query_status: string }`

### `GET /api/proxy/patch/installations`

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `subscriptions` | string (comma-separated) | yes | Subscription IDs to query |

**Response:** `{ installations: Installation[], total_count: number, days: number, query_status: string }`

Both proxy routes follow the exact pattern of `app/api/proxy/incidents/route.ts`:
- `getApiGatewayUrl()` + `buildUpstreamHeaders(auth, false)` + `AbortSignal.timeout(15000)`
- Error envelope: `{ error: string }`
- 502 on gateway unreachable
