# Phase 9 — UI Design Contract

> Visual and interaction contract for the complete Web UI rebuild. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** Tear down all Fluent UI / Griffel styling and replace with Tailwind CSS v4 + shadcn/ui. Redesign every component for a modern enterprise SaaS aesthetic. All business logic (SSE, hooks, MSAL auth, approval flow) is preserved; only the visual layer changes.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (latest, 2026) |
| Preset | New York (closest to enterprise SaaS aesthetic) |
| Component library | Radix UI (via shadcn/ui primitives) |
| Icon library | lucide-react |
| Font | Inter (headings + body) via `next/font/google` |
| CSS framework | Tailwind CSS v4 |
| Markdown renderer | react-markdown + remark-gfm (kept) |
| Panel resize | react-resizable-panels (kept) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1`, `gap-1` | Icon gaps, inline padding |
| sm | 8px | `p-2`, `gap-2` | Compact element spacing, badge padding |
| md | 16px | `p-4`, `gap-4` | Default element spacing, card padding |
| lg | 24px | `p-6`, `gap-6` | Section padding, panel padding |
| xl | 32px | `p-8`, `gap-8` | Layout gaps, major section separation |
| 2xl | 48px | `p-12`, `gap-12` | Page-level vertical breathing room |
| 3xl | 64px | `p-16`, `gap-16` | Hero/empty-state vertical centering |

Exceptions: Touch targets for interactive elements (buttons, chips) use minimum 36px height (`h-9`) for accessibility. Send button uses 40px (`h-10`).

## Typography

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|------------------|
| Caption | 12px | 400 (regular) | 1.33 | `text-xs font-normal` |
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal` |
| Label | 14px | 500 (medium) | 1.5 | `text-sm font-medium` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold` (unused unless marketing page added) |

Font stack: `Inter, ui-sans-serif, system-ui, -apple-system, sans-serif`

Monospace (code blocks, resource IDs): `"JetBrains Mono", ui-monospace, monospace` at 13px (`text-[13px] font-mono`)

## Color

### Light Mode (Primary)

| Role | Hex | Tailwind Token | Usage |
|------|-----|----------------|-------|
| Background (60%) | `#FFFFFF` | `bg-background` | Page background, chat panel bg, card bg |
| Muted surface (30%) | `#F4F4F5` | `bg-muted` | Dashboard panel bg, sidebar bg, input bg, table header bg |
| Azure accent (10%) | `#0078D4` | `bg-primary` | Primary buttons, send button, active tab indicator, user bubble bg, links |
| Azure accent hover | `#106EBE` | `hover:bg-primary/90` | Primary button hover |
| Azure accent light | `#50A7FF` | `text-primary/70` | Active tab text, breadcrumb links |
| Foreground | `#09090B` | `text-foreground` | Body text, headings |
| Muted foreground | `#71717A` | `text-muted-foreground` | Timestamps, captions, placeholder text, secondary labels |
| Border | `#E4E4E7` | `border-border` | Card borders, input borders, separator lines, resize handle |
| Card | `#FFFFFF` | `bg-card` | Chat bubbles, metric cards, proposal cards |
| Destructive | `#DC2626` | `bg-destructive` | Reject button, critical badges, error states |
| Destructive foreground | `#FFFFFF` | `text-destructive-foreground` | Text on destructive buttons |
| Success | `#16A34A` | `text-green-600` | Success badges, healthy status indicators |
| Warning | `#CA8A04` | `text-yellow-600` | Warning badges, degraded status indicators |

### Dark Mode (Optional future — not in Phase 9 MVP)

Dark mode is deferred. The shadcn/ui `dark` class toggle architecture will be included in the Tailwind config so it can be enabled later without structural changes.

### Accent Reserved For

- Primary action buttons (Send Message, Approve Action, Export Report)
- User message bubbles (background)
- Active tab underline / indicator
- Link text
- Focus ring (`ring-primary`)
- Agent name badge (subtle variant)

### Chat Panel Critical Scroll Layout (EXACT implementation)

```tsx
{/* Outer container — fills panel absolutely */}
<div className="absolute inset-0 flex flex-col overflow-hidden">

  {/* Messages area — scrollable, takes remaining space */}
  <ScrollArea className="flex-1 min-h-0">
    <div className="flex flex-col px-4 py-4">
      {/* Chat bubbles render here */}
    </div>
  </ScrollArea>

  {/* Input area — pinned to bottom, never scrolls */}
  <div className="shrink-0 grow-0">
    {/* Example chips */}
    <div className="flex flex-wrap gap-2 px-4 py-1">
      {/* chips */}
    </div>
    {/* Input row */}
    <div className="flex items-end gap-2 px-4 py-3 border-t border-border">
      <Textarea ... />
      <Button ...>Send</Button>
    </div>
  </div>

</div>
```

**Why this works with Tailwind (and why Griffel failed):**
- `flex flex-col` + `flex-1 min-h-0` on the messages area is a single-specificity utility that cannot be overridden by atomic CSS conflicts
- `shrink-0 grow-0` on the input area prevents flex compression
- No CSS-in-JS runtime; classes are compiled at build time; no specificity fights
- `overflow-hidden` on the outer container prevents any content from escaping
