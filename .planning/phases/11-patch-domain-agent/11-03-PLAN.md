## Goal

Add the Patch agent to the Terraform infrastructure (Container App, RBAC, env vars) and CI/CD pipeline (Docker image build). The patch agent Container App gets a system-assigned identity, Reader + Monitoring Reader RBAC on all subscriptions (for ARG cross-subscription queries), the `PATCH_AGENT_ID` env var is wired into the orchestrator, and the deploy-all-images workflow builds the patch agent image.

<action>
Modify `terraform/modules/rbac/main.tf` to add RBAC assignments for the patch agent.

The patch agent needs:
- **Reader** on all in-scope subscriptions — required for ARG cross-subscription queries (ARG inherits RBAC)
- **Monitoring Reader** on all in-scope subscriptions — required for Log Analytics ConfigurationData queries

**Precise insertion point (ISSUE-04 fix):** Insert the patch `merge(...)` block as a new top-level argument to the outer `merge()` call in `local.role_assignments`. The insertion point is:
- **After** the closing `},` on approximately line 99 that ends the Arc Agent `for rg_id in var.arc_resource_group_ids` block
- **Before** the `# API Gateway:` comment on approximately line 101

In the current file, the structure around the insertion point looks like:
```hcl
    # Arc Agent: Contributor on Arc resource groups ...
    {
      for rg_id in var.arc_resource_group_ids :
      "arc-contributor-${md5(rg_id)}" => {
        ...
      }
    },
                        ← INSERT THE PATCH BLOCK HERE (as a new merge() argument)
    # API Gateway: Cognitive Services User on platform subscription ...
    {
      "api-gateway-coguser-platform" = {
```

Insert this block at that position:

```hcl
    # Patch Agent: Reader + Monitoring Reader across all in-scope subscriptions (ARG cross-subscription queries)
    merge(
      {
        for sub_id in var.all_subscription_ids :
        "patch-reader-${replace(sub_id, "-", "")}" => {
          principal_id         = var.agent_principal_ids["patch"]
          role_definition_name = "Reader"
          scope                = "/subscriptions/${sub_id}"
        }
      },
      {
        for sub_id in var.all_subscription_ids :
        "patch-monreader-${replace(sub_id, "-", "")}" => {
          principal_id         = var.agent_principal_ids["patch"]
          role_definition_name = "Monitoring Reader"
          scope                = "/subscriptions/${sub_id}"
        }
      }
    ),
```

This follows the exact same pattern as the SRE Agent RBAC block (Reader + Monitoring Reader across all subscriptions). The `merge()` inner call is needed because there are two `for` comprehensions producing separate maps that must be flattened into the outer `merge()`.

No changes to `variables.tf` — the `agent_principal_ids` map automatically includes the patch agent's principal_id because it's emitted by the agent-apps module's `for_each` over `local.agents`.
</action>
