# Phase 5 — UI Design Contract

> Visual and interaction contract for the Triage & Remediation Web UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Design system:** Fluent UI v9 (`@fluentui/react-components` 9.73.4) with Griffel CSS-in-JS. No shadcn, no Tailwind. All tokens derive from Fluent v9's `tokens` object (`tokens.spacingHorizontalM`, `tokens.fontSizeBase300`, etc.) to guarantee consistency with the Microsoft design language.

## Spacing Scale

Fluent UI v9 uses a 4-point base spacing system via `tokens.spacingHorizontal*` and `tokens.spacingVertical*`. All spacing in Phase 5 components MUST use these tokens — never raw pixel values.

| Token | Fluent v9 Token | Value | Usage |
|-------|----------------|-------|-------|
| xs | `tokens.spacingHorizontalXS` | 4px | Icon-to-label gaps, inline badge padding |
| sNudge | `tokens.spacingHorizontalSNudge` | 6px | Tight icon button internal padding |
| s | `tokens.spacingHorizontalS` | 8px | Chat bubble internal padding (horizontal), compact list item gaps |
| mNudge | `tokens.spacingHorizontalMNudge` | 10px | Input field internal padding |
| m | `tokens.spacingHorizontalM` | 12px | Default element spacing, card content padding |
| l | `tokens.spacingHorizontalL` | 16px | Section padding, chat panel gutter, tab content padding |
| xl | `tokens.spacingHorizontalXL` | 20px | Panel internal margins |
| xxl | `tokens.spacingHorizontalXXL` | 24px | Major section padding, split-pane gutter |
| xxxl | `tokens.spacingHorizontalXXXL` | 32px | Page-level padding, layout outer margins |

**Exceptions:**
- Split-pane resize handle: 8px wide (1px visible border + 3.5px hit area each side) — uses `tokens.spacingHorizontalS` for the handle width
- Touch targets: minimum 32px height for all interactive elements (Fluent v9 enforces this by default on `Button`, `MenuItem`, etc.)

**Source:** Fluent UI v9 spacing tokens (default values), RESEARCH.md §2.3

## Typography

Fluent UI v9 typography uses the `typographyStyles` presets. Phase 5 uses exactly 4 roles mapped to Fluent v9 presets — no custom font sizes.

| Role | Fluent v9 Preset | Size | Weight | Line Height | Usage |
|------|-----------------|------|--------|-------------|-------|
| Body | `typographyStyles.body1` | 14px | Regular (400) | 20px (1.43) | Chat messages, alert descriptions, table cell text, form labels |
| Caption | `typographyStyles.caption1` | 12px | Regular (400) | 16px (1.33) | Timestamps, secondary metadata, badge labels, similarity scores |
| Subtitle | `typographyStyles.subtitle2` | 16px | Semibold (600) | 22px (1.375) | Card titles, panel headers, agent names, tab labels |
| Title | `typographyStyles.title3` | 20px | Semibold (600) | 28px (1.4) | Page title ("Azure AIOps"), section headings |

**Constraints:**
- Maximum 2 weights: Regular (400) and Semibold (600). No Bold (700), no Light (300).
- All text uses `tokens.fontFamilyBase` (Segoe UI Variable). No monospace except within `TraceTree` code blocks, which use `tokens.fontFamilyMonospace` (Cascadia Code, Consolas fallback).
- Body line-height 1.43 (Fluent v9 default for 14px) — not 1.5. Fluent v9 optimizes line heights per size.

**Source:** Fluent UI v9 typography styles (body1, caption1, subtitle2, title3), RESEARCH.md §2.4

## Color

Fluent UI v9 manages color via semantic design tokens. Phase 5 uses the `webLightTheme` palette with dark mode support via `webDarkTheme`. All colors MUST reference Fluent v9 tokens — never raw hex values in component code.

| Role | Fluent v9 Token | Light Value | Usage |
|------|----------------|-------------|-------|
| Dominant (60%) | `tokens.colorNeutralBackground1` | #FFFFFF | Page background, chat panel background, main content area |
| Secondary (30%) | `tokens.colorNeutralBackground3` | #F5F5F5 | Dashboard panel background, card surfaces, alert feed background, sidebar |
| Accent (10%) | `tokens.colorBrandBackground` | #0F6CBD (Brand Blue) | **Reserved for:** Primary CTA buttons ("Send Message", "Approve"), active tab indicator, selected subscription chips, chat input submit button, unread alert count badge |
| Destructive | `tokens.colorPaletteRedBackground3` | #C4314B | **Reserved for:** "Reject" button on proposal cards, critical severity badge, error state icons |

**Semantic status colors (alert severity badges and status indicators only):**

| Status | Fluent v9 Token | Usage |
|--------|----------------|-------|
| Success | `tokens.colorPaletteGreenBackground2` | Resolved/Executed status badge, healthy resource indicator |
| Warning | `tokens.colorPaletteYellowBackground2` | Sev2/Sev3 alert badge, pending approval status |
| Danger | `tokens.colorPaletteRedBackground2` | Sev0/Sev1 alert badge, failed/aborted status |
| Info | `tokens.colorPaletteBlueBorderActive` | Informational status, "Acknowledged" state |

**Accent reserved-for list (exhaustive):**
1. "Send Message" button in chat input
2. "Approve" button on remediation proposal cards
3. Active tab underline in dashboard right-pane
4. Selected subscription chips in multi-subscription picker
5. Unread alert count badge
6. Primary navigation selected state
7. Chat input focus ring

No other elements may use `colorBrandBackground`. Secondary actions use `tokens.colorNeutralBackground1` with `tokens.colorNeutralForeground2` text (ghost button pattern).

**Source:** Fluent UI v9 webLightTheme tokens, CONTEXT.md D-02, RESEARCH.md §2.4

## Component Inventory

All Phase 5 custom components built on Fluent v9 primitives. No third-party UI component registries.

### Chat Panel Components

| Component | Fluent v9 Base | Props | Behavior |
|-----------|---------------|-------|----------|
| `ChatBubble` | `Card` variant="subtle" + `Text` + `makeStyles` | `agentName: string`, `content: string`, `isStreaming: boolean`, `timestamp: string` | Left-aligned for agent messages. When `isStreaming=true`, appends characters from `event:token` deltas with a blinking cursor (CSS `@keyframes blink`). Shows agent name in `Caption` above bubble. Background: `tokens.colorNeutralBackground3`. |
| `UserBubble` | `Card` variant="filled" + `Text` | `content: string`, `timestamp: string` | Right-aligned. Background: `tokens.colorBrandBackground`. Text: `tokens.colorNeutralForegroundOnBrand`. |
| `AgentAvatar` | `Avatar` + `Persona` | `agentName: string`, `domain: string` | 32px avatar with domain-specific icon. Name displayed via `Persona` in `Caption` weight. |
| `ThinkingIndicator` | `Spinner` size="tiny" + `Text` variant="caption1" | `agentName: string` | "Compute Agent is thinking..." with Fluent `Spinner` animation. Appears during handoff gaps (between agent responses). Disappears when next `event:token` arrives. |
| `ChatInput` | `Textarea` + `Button` | `onSend: (message: string) => void`, `disabled: boolean` | Single-line expanding textarea (max 4 rows). Send button uses `tokens.colorBrandBackground` with `SendRegular` icon. Submit on Enter, Shift+Enter for newline. Disabled during streaming. |
| `ProposalCard` | `Card` + `Button` + `Badge` + `Text` | `proposal: ApprovalRecord`, `onApprove: () => void`, `onReject: () => void` | Embedded in chat stream at approval gate marker. Shows: action description (`Body`), target resources (`Caption` monospace), impact (`Body`), risk level (`Badge` — "High" uses `tokens.colorPaletteRedBackground2`, "Critical" uses `tokens.colorPaletteRedBackground3`), expiry countdown timer (live `Caption` text updating every second). Two buttons: "Approve" (`tokens.colorBrandBackground`), "Reject" (`tokens.colorPaletteRedBackground3`). Buttons disabled when `status !== "pending"`. |

### Trace Panel Components (UI-004)

| Component | Fluent v9 Base | Props | Behavior |
|-----------|---------------|-------|----------|
| `TraceTree` | `Tree` + `TreeItem` + `TreeItemLayout` | `events: TraceEvent[]` | Expandable tree. Collapsed by default. Each node shows: event type icon + tool/agent name + duration badge. Expanding shows: full JSON payload in `tokens.fontFamilyMonospace` 12px. Event types: `tool_call` (wrench icon), `handoff` (arrow icon), `approval_gate` (shield icon). |
| `TraceEventNode` | `TreeItem` + `Badge` + `Text` | `event: TraceEvent` | Single tree node. Status badge: success=green, error=red, pending=yellow. Duration shown as `Caption` text (e.g., "234ms"). |

### Dashboard Components

| Component | Fluent v9 Base | Props | Behavior |
|-----------|---------------|-------|----------|
| `AlertFeed` | `DataGrid` + `DataGridRow` + `DataGridCell` + `Badge` | `incidents: IncidentSummary[]`, `filters: FilterState` | Columns: Severity (badge), Domain, Resource, Status, Time. Severity badges use semantic colors (Sev0/1=red, Sev2/3=yellow, Sev4=blue). Real-time updates via polling (5s interval). Clicking a row opens the incident in the chat panel. |
| `AlertFilters` | `Toolbar` + `Dropdown` + `SearchBox` | `onFilterChange: (filters: FilterState) => void` | Filter controls: subscription (multi-select), severity (multi-select), domain (multi-select), status (single-select: All/New/Acknowledged/Closed). Horizontal toolbar above `AlertFeed`. |
| `SubscriptionSelector` | `Combobox` multiselect + `Tag` | `subscriptions: Subscription[]`, `selected: string[]`, `onChange: (ids: string[]) => void` | Multi-select subscription picker in the top navigation bar. Selected subscriptions shown as `Tag` chips. Scopes all dashboard views and agent queries (UI-007). |
| `AuditLogViewer` | `DataGrid` + `Toolbar` + `DatePicker` | `filters: AuditFilterState` | Columns: Timestamp, Agent, Tool, Action, Resource, Outcome, Duration. Filters: agent name, action type, resource ID, time range (DatePicker). Data from `GET /api/v1/audit`. Pagination: 50 rows per page. |
| `ResourcesGrid` | `DataGrid` + `Badge` | `resources: Resource[]` | Resource inventory grid. Columns: Name, Type, Subscription, Status, Region. Status badge colors per semantic status tokens. |

## Interaction Patterns

### Chat Streaming (UI-003)

| State | Visual | Trigger |
|-------|--------|---------|
| Idle | Chat input enabled, no indicators | No active conversation or stream complete |
| Streaming | `ChatBubble` with `isStreaming=true` (blinking cursor), `ChatInput` disabled | `event:token` SSE events arriving |
| Thinking (handoff) | `ThinkingIndicator` with next agent name | Gap between last `event:token` and next; `event:trace` shows `handoff_start` |
| Approval gate | `ProposalCard` embedded in chat, streaming paused | `event:trace` with `type: "approval_gate"` |
| Reconnecting | Fluent `MessageBar` intent=warning at top of chat: "Reconnecting..." | EventSource `onerror` fired |
| Reconnected | MessageBar disappears, missed events replayed | EventSource `onopen` after reconnect |
| Error | Fluent `MessageBar` intent=error: error state copy (see Copywriting) | `event:error` or 3 failed reconnect attempts |

### Approval Flow (UI-005, REMEDI-005)

| State | ProposalCard Visual | Buttons |
|-------|-------------------|---------|
| Pending | Expiry countdown visible, risk badge | "Approve" (brand), "Reject" (destructive) — both enabled |
| Approved | Green `Badge` "Approved", decided_by name, timestamp | Both disabled, muted |
| Rejected | Red `Badge` "Rejected", decided_by name, timestamp | Both disabled, muted |
| Expired | Yellow `Badge` "Expired", no countdown | Both disabled, muted |
| Aborted | Red `Badge` "Aborted — stale_approval", abort reason text | Both disabled, muted |

### Alert Feed (UI-006)

| State | Visual |
|-------|--------|
| Loading | Fluent `Skeleton` shimmer rows (5 rows) |
| Populated | `DataGrid` rows, new alerts animate in with `tokens.durationNormal` (200ms) fade |
| Empty | Empty state (see Copywriting) |
| Filtering | Instant client-side filter; grid re-renders without loading state |
| New alert arrives | Row prepends with highlight background (`tokens.colorNeutralBackground1Hover`) fading to normal over 2s |

## Animation & Motion

All animations use Fluent v9 motion tokens. No custom keyframes except the streaming cursor blink.

| Element | Token/Duration | Easing |
|---------|---------------|--------|
| Alert row enter | `tokens.durationNormal` (200ms) | `tokens.curveEasyEase` |
| Alert row highlight fade | 2000ms (custom) | linear fade from `colorNeutralBackground1Hover` to `colorNeutralBackground1` |
| Tab switch | `tokens.durationFast` (100ms) | `tokens.curveEasyEase` |
| Proposal card expand | `tokens.durationNormal` (200ms) | `tokens.curveEasyEase` |
| Trace tree expand/collapse | `tokens.durationNormal` (200ms) | `tokens.curveEasyEase` |
| Streaming cursor blink | 530ms on / 530ms off | step-end (CSS `@keyframes blink { 50% { opacity: 0; } }`) |
| Thinking spinner | Fluent `Spinner` built-in animation | Built-in |
| MessageBar slide-in | `tokens.durationNormal` (200ms) | `tokens.curveEasyEaseMax` |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | All interactive elements reachable via Tab. Chat input: Enter to send, Shift+Enter for newline. Approve/Reject buttons: Enter or Space to activate. Tab navigation in dashboard: Arrow keys between tabs. |
| Screen reader | Chat messages use `role="log"` with `aria-live="polite"`. New alerts use `aria-live="polite"`. Proposal cards use `role="alertdialog"` with descriptive `aria-label`. |
| Focus management | After sending a message, focus returns to chat input. After approve/reject, focus moves to next actionable element. On tab switch, focus moves to first focusable element in tab content. |
| Color contrast | Fluent v9 tokens guarantee WCAG 2.1 AA contrast ratios (4.5:1 for text, 3:1 for UI components) in both light and dark themes. |
| Reduced motion | Respect `prefers-reduced-motion: reduce`. Disable streaming cursor blink, alert row animation, and expand/collapse transitions. Spinner uses static indicator. |

## Dark Mode

Phase 5 supports dark mode via Fluent v9 `webDarkTheme`. Implementation:

- `FluentProvider` accepts `theme` prop: `webLightTheme` or `webDarkTheme`.
- Toggle button in top navigation bar using `WeatherMoonRegular` / `WeatherSunnyRegular` icons.
- Theme preference persisted to `localStorage` key `aap-theme`.
- All component colors derive from Fluent tokens — no raw hex values — so theme switching is automatic.
- No additional CSS or token overrides needed for dark mode.
