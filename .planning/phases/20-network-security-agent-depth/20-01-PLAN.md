<threat_model>
## Threat Model (ASVS L1)

### Trust Boundaries
- Network agent tools run inside a Container App with system-assigned managed identity
- Network Contributor RBAC scoped to network subscription only (enforced by Terraform)
- Tools make read-only SDK calls except `check_connectivity` which triggers a Network Watcher diagnostic (non-destructive)

### Identified Threats
1. **Over-permissioned SDK calls:** Tools could be used to invoke write operations on NSGs/VNets
   - Mitigation: All tools are read-only queries (GET operations). `check_connectivity` is a diagnostic, not a mutation. RBAC enforced by Terraform at Network Contributor level.
2. **Subscription ID injection:** Malicious resource_id could target unauthorized subscriptions
   - Mitigation: `_extract_subscription_id()` extracts from resource_id format only; RBAC prevents access to out-of-scope subscriptions via 403.
3. **LRO timeout abuse:** `check_connectivity` LRO could hang indefinitely
   - Mitigation: 120-second timeout on `poller.result()`. Returns timeout error dict, never blocks.
4. **Information leakage via error messages:** SDK exceptions could expose internal details
   - Mitigation: Error handler returns `str(e)` only; structured error dicts, not stack traces.

### Residual Risk
- **Low:** Network Watcher diagnostic requires the extension on source VM; may fail silently if not installed. Acceptable — tool returns error dict.
</threat_model>

- On timeout: return `{"query_status": "timeout", "error": "Connectivity check timed out after 120 seconds"}`
