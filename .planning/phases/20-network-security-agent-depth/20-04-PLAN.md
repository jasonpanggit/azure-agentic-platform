## Goal

Create comprehensive unit tests for all 21 tools across the 3 agents (7 network + 7 security + 7 SRE). Each test file follows the established `test_patch_tools.py` pattern: mock at SDK client level, test success path, error path, and SDK-missing path for every tool. Extend integration tests with 3 new triage flow scenarios. Target: >=80% coverage per agent's `tools.py`.

<threat_model>
## Threat Model (ASVS L1)

### Trust Boundaries
- Tests run in CI (GitHub Actions) and developer machines
- Tests mock all Azure SDK clients — no real Azure calls
- Test fixtures contain synthetic data only

### Identified Threats
1. **Test data leakage:** Hardcoded subscription IDs or resource IDs in tests could be real values
   - Mitigation: All test data uses obviously fake IDs: `sub-test-1`, `rg-test`, `nsg-test`, etc.
2. **Flaky tests due to import order:** Mocking at module level could cause test isolation issues
   - Mitigation: Each test uses `@patch` decorators that are scoped per-test. No module-level state mutation.

### Residual Risk
- **None:** Tests are read-only, execute no Azure calls, and contain no secrets.
</threat_model>
