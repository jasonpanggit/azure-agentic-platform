<threat_model>
## Threat Model (ASVS L1)

### Trust Boundaries
- Security agent runs inside a Container App with system-assigned managed identity
- Security Reader RBAC scoped across all in-scope subscriptions (enforced by Terraform)
- Tools are strictly read-only — no write operations on RBAC, policies, or Defender
- Key Vault tool queries control-plane logs ONLY — never accesses data plane (secrets/keys/certificates)

### Identified Threats
1. **Key Vault data plane access:** Tool could be modified to access secrets
   - Mitigation: `query_keyvault_diagnostics` uses Log Analytics KQL (`AzureDiagnostics` table) — never touches Key Vault REST API data plane. RBAC is Security Reader, not Key Vault access.
2. **RBAC enumeration exposure:** `query_rbac_assignments` could expose sensitive role assignments
   - Mitigation: Tool returns role assignments visible to the Security Reader role only. No elevation possible. Data flows only to the agent, not externally.
3. **Large policy compliance result sets causing OOM:** Subscription with thousands of non-compliant resources
   - Mitigation: Results capped at top 100 non-compliant resources via iteration limit.
4. **Subscription ID parameter injection:** Could target unauthorized subscriptions
   - Mitigation: RBAC prevents access to out-of-scope subscriptions (403). All tools accept explicit `subscription_id` parameter.
5. **New SDK packages introduce supply chain risk**
   - Mitigation: All packages are official Microsoft Azure SDK packages (`azure-mgmt-*`), pinned with minimum versions.

### Residual Risk
- **Low:** SecurityCenter requires `asc_location` parameter; wrong value returns empty results but no error. Default "centralus" covers most deployments.
</threat_model>

**Tool 1: `query_defender_alerts(subscription_id, severity=None, asc_location="centralus")`**
- Add `asc_location: str = "centralus"` parameter
- SDK: `SecurityCenter(credential, subscription_id, asc_location=asc_location).alerts.list()`
- For each alert: `alert_display_name`, `severity`, `status`, `description`, `compromised_entity`, `time_generated_utc` (isoformat if not None), `alert_type`, `product_name`
- Filter: if `severity` is not None, only include alerts matching severity
- Return: `{"subscription_id": ..., "severity_filter": severity, "asc_location": asc_location, "alerts": [...], "alert_count": len(alerts), "query_status": "success"}`
- Cap at 200 alerts max to prevent OOM

**Tool 2: `query_keyvault_diagnostics(vault_name, timespan_hours=2, workspace_id=None)`**
- Add `workspace_id: Optional[str] = None` parameter. If None, read from `LOG_ANALYTICS_WORKSPACE_ID` env var. If still None, return error.
- SDK: `LogsQueryClient(credential).query_workspace(workspace_id, query, timespan=timedelta(hours=timespan_hours))`
- KQL query:
  ```
  AzureDiagnostics
  | where ResourceProvider == "MICROSOFT.KEYVAULT"
  | where Resource =~ "{vault_name}"
  | where TimeGenerated > ago({timespan_hours}h)
  | project TimeGenerated, OperationName, CallerIPAddress, ResultType, ResultSignature, ResultDescription, Identity=identity_claim_upn_s
  | order by TimeGenerated desc
  | take 100
  ```
- For each row: extract column values into dict
- Anomaly detection: flag operations from new IPs (> 5 unique callers), failed operations (ResultType != "Success"), bulk operations (> 20 same-type ops)
- Return: `{"vault_name": ..., "timespan_hours": ..., "workspace_id": ..., "operations": [...], "operation_count": int, "anomaly_indicators": [...], "query_status": "success"}`

**Tool 3: `query_iam_changes(subscription_id, timespan_hours=2)`**
- SDK: `MonitorManagementClient(credential, subscription_id).activity_logs.list(filter=filter_str)`
- Filter: `"eventTimestamp ge '{start.isoformat()}' and resourceProvider eq 'Microsoft.Authorization'"` where start is now - timespan_hours
- Categorize events into:
  - `rbac_changes`: operations containing `roleAssignments/write` or `roleAssignments/delete`
  - `keyvault_policy_changes`: operations containing `Microsoft.KeyVault/vaults/accessPolicies`
  - `identity_operations`: operations containing `Microsoft.ManagedIdentity` or `Microsoft.Authorization/policyAssignments`
- For each event: `eventTimestamp` (isoformat), `operationName`, `caller`, `status`, `resourceId`, `level`, `description`
- Return: `{"subscription_id": ..., "timespan_hours": ..., "rbac_changes": [...], "keyvault_policy_changes": [...], "identity_operations": [...], "total_changes": int, "query_status": "success"}`

- SDK: If `scope` is provided: `AuthorizationManagementClient(credential, subscription_id).role_assignments.list_for_scope(scope=scope)`. Else: `client.role_assignments.list_for_subscription()`
- Filter: if `principal_id` is not None, filter results where `ra.principal_id == principal_id`
- For each assignment: `id`, `principal_id`, `principal_type`, `role_definition_id`, `scope`, `created_on` (isoformat if not None), `updated_on` (isoformat if not None)
- Cap at 500 assignments max
- Return: `{"subscription_id": ..., "scope_filter": scope, "principal_id_filter": principal_id, "assignments": [...], "assignment_count": int, "query_status": "success"}`

**Tool 6: `query_policy_compliance(subscription_id, compliance_state="NonCompliant", max_results=100)`**
- SDK: `PolicyInsightsClient(credential, subscription_id).policy_states.list_query_results_for_subscription(policy_states_resource="latest")`
- Filter in-code: only include results where `state.compliance_state == compliance_state`
- For each state (up to `max_results`): `resource_id`, `policy_assignment_id`, `policy_definition_id`, `compliance_state`, `resource_type`, `resource_group`, `is_compliant` (bool), `policy_definition_action`
- Return: `{"subscription_id": ..., "compliance_state_filter": compliance_state, "max_results": max_results, "policy_states": [...], "non_compliant_count": int, "query_status": "success"}`

**Tool 7: `scan_public_endpoints(subscription_id)`**
- SDK: `NetworkManagementClient(credential, subscription_id).public_ip_addresses.list_all()`
- For each public IP: `name`, `resource_group` (extract from id), `ip_address`, `public_ip_allocation_method`, `ip_configuration_id` (from `ip_configuration.id` if present), `dns_fqdn` (from `dns_settings.fqdn` if present), `sku_name` (from `sku.name`), `associated` (bool — True if ip_configuration is not None)
- Return: `{"subscription_id": ..., "public_ips": [...], "public_ip_count": int, "associated_count": int (where ip_configuration is not None), "unassociated_count": int, "query_status": "success"}`
