# 10-02: Runbook Search Availability Hardening

> Wave 2 remediation. Harden the runbook search data-plane contract so misconfiguration or DB outages do not surface as generic 500s.

## Goal

Make runbook search resolve the correct DSN contract, fail fast when the DB is not configured, and return 503 for runbook-data-plane unavailability.

---

## Tasks

<task id="10-02-01">
<title>Make runbook DB configuration explicit</title>
<read_first>
- services/api-gateway/runbook_rag.py
- services/api-gateway/main.py
</read_first>
<action>
1. Add a shared DSN resolver for runbook search.
2. Support `PGVECTOR_CONNECTION_STRING` as an alias for the runbook database DSN.
3. Stop implicitly falling back to `localhost` when no runbook DB configuration is present.
</action>
<acceptance_criteria>
- runbook search accepts `PGVECTOR_CONNECTION_STRING`
- runbook search accepts `POSTGRES_DSN`
- runbook search raises a typed availability error when no runbook DB configuration is present
</acceptance_criteria>
</task>

<task id="10-02-02">
<title>Translate runbook data-plane failures into 503s</title>
<read_first>
- services/api-gateway/main.py
- services/api-gateway/tests/test_runbook_search_availability.py
</read_first>
<action>
1. Add a dedicated `RunbookSearchUnavailableError` boundary in the runbook search module.
2. Catch that error in `/api/v1/runbooks/search` and return HTTP 503 with a truthful message.
3. Reuse the same resolver in startup migrations so configuration logic is consistent.
</action>
<acceptance_criteria>
- `/api/v1/runbooks/search` returns 503 instead of 500 when the runbook DB is unavailable
- startup migrations skip cleanly when the runbook DB is not configured
</acceptance_criteria>
</task>
