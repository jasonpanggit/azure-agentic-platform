# 10-01: API Gateway Auth & Audit Hardening

> Wave 1 remediation. Tight, code-only hardening pass that removes fail-open gateway auth by omission and blocks unsafe audit-query inputs.

## Goal

Make the API gateway fail closed unless insecure local auth is explicitly enabled, and validate audit query inputs before any KQL is built or executed.

---

## Tasks

<task id="10-01-01">
<title>Make gateway auth fail closed by default</title>
<read_first>
- services/api-gateway/auth.py
- services/api-gateway/tests/conftest.py
- scripts/run-mock.sh
</read_first>
<action>
1. Replace the current implicit auth bypass in `services/api-gateway/auth.py` with an explicit mode switch.
2. Default to Entra validation mode.
3. Only allow unauthenticated local requests when `API_GATEWAY_AUTH_MODE=disabled` is set.
4. When auth is enabled but required Entra configuration is missing, return a truthful 503 instead of silently authorizing requests.
5. Update test and local mock bootstrap so they opt into disabled auth explicitly.
</action>
<acceptance_criteria>
- `services/api-gateway/auth.py` does NOT authorize requests merely because `AZURE_CLIENT_ID` is unset
- `services/api-gateway/auth.py` supports `API_GATEWAY_AUTH_MODE=disabled` for explicit local bypass
- Missing auth config in enabled mode returns a 503 configuration error
- `scripts/run-mock.sh` sets `API_GATEWAY_AUTH_MODE=disabled`
- gateway tests continue to run under explicit disabled mode
</acceptance_criteria>
</task>

<task id="10-01-02">
<title>Validate audit filters before building KQL</title>
<read_first>
- services/api-gateway/audit.py
- services/api-gateway/main.py
- services/api-gateway/tests/test_audit_trail.py
</read_first>
<action>
1. Add input validation helpers for `agent`, `action`, `incident_id`, `resource`, `from_time`, `to_time`, and `limit`.
2. Reject malformed or unsafe values before KQL construction.
3. Convert audit route input validation failures into HTTP 400 responses.
4. Preserve existing valid KQL filters for normal queries.
</action>
<acceptance_criteria>
- invalid audit filters raise `ValueError` before any `LogsQueryClient.query_workspace()` call
- `/api/v1/audit` returns 400 for invalid query parameters
- existing valid audit filter tests still pass
- KQL is still generated for valid agent and time-range filters
</acceptance_criteria>
</task>
