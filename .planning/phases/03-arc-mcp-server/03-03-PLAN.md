# Plan 03-03: Unit Tests + CI

> **Wave 2** — Depends on 03-01 (server.py, models.py, tools/ must exist to test).

## must_haves

1. `test_pagination.py` seeds exactly 120 mock Arc servers and asserts `total_count == 120` and `len(servers) == 120` — this is the direct unit-level proof of AGENT-006.
2. `test_arc_servers.py` tests `_is_prolonged_disconnect` with three cases: `status == "Connected"` (returns False), `status == "Disconnected"` with `last_status_change` within threshold (returns False), and `status == "Disconnected"` with `last_status_change` exceeding threshold (returns True) — MONITOR-004 logic.
3. `test_arc_servers.py` tests `arc_extensions_list_impl` with a mocked extension set including one AMA extension with `provisioning_state == "Succeeded"` and one Change Tracking extension with `provisioning_state == "Failed"` — verifies MONITOR-005 field mapping.
4. `test_arc_k8s.py` tests `arc_k8s_gitops_status_impl` with a mocked `SourceControlConfigurationClient` — one cluster with two Flux configurations (compliance_state "Compliant" and "NonCompliant"), asserts `flux_detected == True` and `total_configurations == 2` (MONITOR-006).
5. `test_arc_k8s.py` tests `arc_k8s_list_impl` with 105 mocked clusters — asserts `total_count == 105` (AGENT-006 for K8s).
6. All tests use `pytest.mark.unit` and never make real Azure API calls — mocking is via `unittest.mock.patch` or `MagicMock` only.
7. CI workflow triggers on changes to `services/arc-mcp-server/**` and runs `pytest -m unit` — fails the build if any test fails.
