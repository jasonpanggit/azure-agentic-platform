# Phase 104-1: Azure Firewall Tab

## Goal
Add an **Azure Firewall** sub-tab to the Resources hub with two sub-views:
1. **Rules Management** ‚Äî list all firewall policies + rules across subscriptions
2. **Rule Audit** ‚Äî identify overlapping rules (consolidation candidates) and overly-wide rules (tightening candidates)

## Requirements
- Live ARG query on mount (no scan button) with 15-min TTL cache
- Two sub-views inside the tab: Rules and Audit
- Rules view: table of all Azure Firewall instances with their policy rules
- Audit view: classified findings (overlap / too-wide) with severity badges and remediation hints
- Follow the NSG audit service pattern for backend; follow NsgAuditTab for UI
- Add `firewall` sub-tab to `ResourcesHubTab` alongside existing sub-tabs

- `services/api-gateway/firewall_service.py` ‚Äî ARG queries + rule classification logic
- `services/api-gateway/firewall_endpoints.py` ‚Äî GET /api/v1/firewall/rules + GET /api/v1/firewall/audit

## Implementation Steps

### Step 1 ‚Äî `firewall_service.py`

ARG KQL for firewall instances + rules:

```kql
Resources
| where type =~ 'microsoft.network/azurefirewalls'
| project
    firewall_id = tolower(id),
    firewall_name = name,
    resourceGroup,
    subscriptionId,
    location,
    sku_tier = tostring(properties.sku.tier),
    threat_intel_mode = tostring(properties.threatIntelMode),
    policy_id = tolower(tostring(properties.firewallPolicy.id))
```

ARG KQL for firewall policy rule collections:

```kql
Resources
| where type =~ 'microsoft.network/firewallpolicies'
| mv-expand rc = properties.ruleCollections
| mv-expand rule = rc.rules
| project
    policy_id = tolower(id),
    policy_name = name,
    resourceGroup,
    subscriptionId,
    collection_name = tostring(rc.name),
    collection_priority = toint(rc.priority),
    action = tostring(rc.action.type),
    rule_name = tostring(rule.name),
    rule_type = tostring(rule['ruleType']),
    source_addresses = tostring(rule.sourceAddresses),
    destination_addresses = tostring(rule.destinationAddresses),
    destination_ports = tostring(rule.destinationPorts),
    protocols = tostring(rule.ipProtocols)
```

Classification logic:
- **Overlap**: two rules with same source+dest+port but different priorities ‚Üí lower priority rule is shadowed
- **Too-wide source**: sourceAddresses contains `*` or `0.0.0.0/0`
- **Too-wide destination**: destinationAddresses contains `*` and destinationPorts contains `*`
- **Too-wide ports**: destinationPorts contains `0-65535` or `*` on Allow rules

Severity mapping:
- `too_wide_source` + internet-facing ‚Üí critical
- `too_wide_destination` ‚Üí high
- `too_wide_ports` ‚Üí high
- `overlap_shadowed` ‚Üí medium (waste / confusion)

Data classes: `FirewallRule`, `FirewallAuditFinding`

Cache: `arg_cache.get_cached("firewall_rules", sub_ids, 900, fetch_fn)`
       `arg_cache.get_cached("firewall_audit", sub_ids, 900, fetch_fn)`

### Step 2 ‚Äî `firewall_endpoints.py`

```python
router = APIRouter(prefix="/api/v1/firewall", tags=["firewall"])

GET /api/v1/firewall/rules
  query params: subscription_ids (comma-sep, optional)
  returns: { firewalls: [...], rules: [...], count: int }

GET /api/v1/firewall/audit
  query params: subscription_ids (comma-sep, optional), severity (optional filter)
  returns: { findings: [...], summary: { critical, high, medium, total }, generated_at }
```
